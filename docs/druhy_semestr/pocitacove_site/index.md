# Počítačové sítě

Předmět Počítačové sítě seznamuje studenty s architekturou, protokoly a technologiemi moderních komunikačních systémů. Výuka je postavena na teoretických základech síťových modelů a jejich praktické aplikaci v reálném prostředí.

## Základní informace

- **Katedra:** Katedra informačních technologií (KTI)
- **Zakončení:** Zápočet + Zkouška
- **Forma výuky:** Přednášky a praktická cvičení v síťové laboratoři

## Obsah předmětu

Kurz se věnuje především hierarchickému uspořádání sítí a klíčovým protokolům:

1. **Referenční modely:** Detailní rozbor modelů OSI (7 vrstev) a TCP/IP.
2. **Fyzická a linková vrstva:** Typy médií (optika, metalika, bezdrát), Ethernet, MAC adresy a přepínání (switching).
3. **Síťová vrstva:** Protokoly IPv4 a IPv6, struktura IP adresy, masky podsítí (subnetting) a základy směrování (routing).
4. **Transportní vrstva:** Rozdíly mezi protokoly TCP (spolehlivý, spojovaný) a UDP (nespolehlivý, datagramový), porty.
5. **Aplikační vrstva:** Principy protokolů HTTP, DNS, DHCP, FTP, SMTP a dalších.
6. **Síťová bezpečnost:** Základy firewallů, NAT (překlad adres) a VPN.

## Podmínky zakončení

### Zápočet

Získání zápočtu obvykle vyžaduje:

- **Laboratorní úlohy:** Praktická konfigurace síťových prvků (routery, switche). Často se využívá simulátor **Cisco Packet Tracer**.
- **Výpočetní testy:** Zaměřené na binární logiku, výpočty masek podsítí (CIDR) a rozsahů IP adres.
- **Docházka:** Povinná účast na měřeních v laboratořích.

### Zkouška

Zkouška prověřuje komplexní znalost fungování sítí:

- **Písemná část:** Teoretické otázky a příklady (např. sestavení routovací tabulky nebo analýza TCP spojení).
- **Ústní část:** Diskuse nad konkrétními protokoly a principy přenosu dat.

## Doporučení a tipy

- **Subnetting:** Výpočty IP adres a masek musíte zvládat "i o půlnoci". Je to základ pro zápočet i zkoušku.
- **Cisco Packet Tracer:** Naučte se s tímto simulátorem pracovat co nejdříve. Umožní vám zkoušet si síťové konfigurace bez rizika poškození reálného hardwaru.
- **Vrstvy:** Neučte se vrstvy OSI modelu jen jako seznam jmen, ale pochopte, co se s daty (PDU - Protocol Data Unit) v každé vrstvě děje (enkapsulace).
- **Wireshark:** Vyzkoušejte si analýzu reálného síťového provozu. Vidět "živé" packety HTTP nebo DNS velmi pomáhá v pochopení teorie.
